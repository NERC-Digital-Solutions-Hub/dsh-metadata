# Landscape point feature data 2007 [Countryside Survey], Great Britain

## Overview
- Landscape point features collected in 2007 from 566 1km squares across Great Britain.
- Dataset version: 1: 04-2-2016; data owned by NERC - Centre for Ecology & Hydrology (CEH).
- Purpose: Provides landscape-scale point feature data to inform environmental monitoring and framework development.

## Data structure and content
- File: LANDSCAPE_PT_FEATURES_2007.csv
- Key fields (examples):
  - SQUARE, YEAR, ID
  - THEME, THEME_NAME
  - PRIMARY_ATTRIBUTE, PRIMARY_ATTRIBUTE_NAME
  - SPECIES, SPECIES_NAME, PROPORTION, PROPORTION_RANGE
  - USE, USE_NAME
  - BUFFER
  - VETERAN_TREE-related: TREE_DEAD, MISSING_LIMBS, DEAD_WOOD, DEAD_MISSING_BARK, LIGHTNING_STRIKES, HOLLOW_TRUNK, MODAL_DBH, MODAL_DBH_RANGE, VETERAN_TREE_TYPE, VETERAN_TREE_TYPE_NAME
  - EPIPHYTE_COVER, EPIPHYTE_COVER_NAME
  - IVY_COVER, IVY_COVER_RANGE
  - CANOPY_LIVE, CANOPY_LIVE_RANGE
  - LC07, LC07_NUM (ITE Land Classification 2007)
  - COUNTRY, COUNTY07
  - EZ_DESC_07 (Environmental Zone description)
- Coding and naming: Many fields use codes (e.g., THEME, SPECIES, CANOPY_LIVE) with human-readable names provided in corresponding *_NAME fields.
- Scope: 1km square sampling; Year indicates survey year; data linked to land use, habitat classifications, veteran-tree attributes, epiphyte and ivy cover.

## Data sources and governance
- Classifications: Draws on ITE Land Classification of Great Britain (Merlewood method) and related biodiversity classifications; links to broader habitat and land-cover frameworks.
- Provenance and references: Documents and methodologies are described on the Countryside Survey website; references include key publications from Bunce et al. (1996, 2007) and Jackson (2000), among others.
- Data management: Emphasises the need for clear metadata, data cleaning/transformation, and transparent data governance to ensure datasets meet standards and can be stored, shared, and updated.

## Usage considerations for monitoring frameworks
- Practical considerations for decision-makers:
  - Data quality and accessibility challenges: potential data gaps, data of insufficient standard, incompatible formats, and data silos between organisations.
  - Metadata adequacy: essential for verifying suitability and reuse.
  - Data sharing and governance: requires processes to ensure openness while respecting rights and provenance.
- Relevance to monitoring:
  - Provides rich, coded attributes for landscape features, including land use, habitat classification, veteran trees, canopy cover, epiphytes, and ivy cover, enabling integrated environmental monitoring and policy evaluation.
